# erli.bg

A learning resource for Sofia-Erli, the Romani dialect spoken by the Erli community of Sofia, Bulgaria. The site offers ten introductory lessons with vocabulary, example sentences, short video clips, and downloadable Anki decks.

The audience is anyone learning Sofia-Erli: community members, linguists, language teachers, social workers, medical staff working in Roma neighbourhoods, and researchers. The materials are non-commercial and open access.

## Deliberate minimalism

The site is plain HTML5. No CSS framework, no JavaScript, no build pipeline required to view a page locally, no webfonts, no images for decoration, no analytics, no third-party scripts. The only external resource on any page is the Vimeo iframe on lesson pages.

The reasons are practical, not stylistic. Many learners in Sofia and elsewhere use older phones on slow or metered connections. Pages need to load in under a second on a 2G link, work in any browser, and remain readable for a decade without maintenance. Plain HTML is the only format that meets those constraints reliably.

Each generated HTML page is well under 5 KB. The only styling is a single inline rule that limits body width to 40em on wide screens. Everything else uses browser defaults: serif body text, blue underlined links, black on white.

## Repository layout

```
/
├── index.html              generated from content/, links every lesson
├── about.html              hand-written
├── sources.html            hand-written
├── lessons/
│   └── lesson-NN.html      generated by build.py
├── content/
│   └── lesson-NN.md        Markdown source with YAML frontmatter
├── audio/                  .mp3 files referenced by the vocab tables
├── apkg/                   Anki decks, one per lesson
├── build.py                ~150 line static-site generator
├── README.md
└── .gitignore
```

## Adding a new lesson

1. `cp content/lesson-10.md content/lesson-11.md`
2. Edit the YAML frontmatter (number, title, vimeo_id, anki_file, recording metadata) and replace the vocabulary and example sentence tables.
3. Drop the audio files into `audio/` and the Anki deck into `apkg/`.
4. Run `python3 build.py` from the repository root.
5. `git add . && git commit -m "Lesson 11: At the pharmacy" && git push`.
6. Cloudflare Pages or GitHub Pages will rebuild and publish within about a minute.

The build script reads every `content/lesson-*.md`, writes the corresponding `lessons/lesson-NN.html`, and regenerates `index.html` so the new lesson appears in the list automatically. The script has no external dependencies beyond a standard Python 3 install.

## Lesson Markdown format

```
---
number: 11
title: At the pharmacy
recorded_at: Fakulteta, Sofia
recorded_on: 2026-07-15
speakers: M.K. and D.A.
vimeo_id: 123456789
anki_file: lesson-11.apkg
---

## Description

Two speakers role-play a visit to a neighbourhood pharmacy.

## Vocabulary

| Romani | English | Audio |
|---|---|---|
| droga | medicine | droga.mp3 |
| nasvalipe | illness | nasvalipe.mp3 |

## Example sentences

| Romani | English | Audio |
|---|---|---|
| Si tu droga vas o sero? | Do you have something for the head? | sentence-01.mp3 |
```

Both tables use the third column for filenames; they are rewritten as plain links into `/audio/` by the build step.

## Deployment

### Cloudflare Pages

1. Push the repository to GitHub or GitLab.
2. In the Cloudflare dashboard, create a new Pages project and connect the repository.
3. Build command: `python3 build.py`
4. Build output directory: `/` (the repository root)
5. Save and deploy. Cloudflare will rebuild on every push to the main branch.

If you would rather precommit the generated HTML and skip the build step in the cloud, leave the build command empty and set the output directory to `/`. The repository already contains the generated HTML.

### GitHub Pages

1. Push the repository to GitHub.
2. In the repository settings, enable Pages from the main branch, root folder.
3. Commit the generated HTML files (run `python3 build.py` locally first).
4. GitHub Pages publishes the static files directly. No build step runs server-side.

For both providers, audio files larger than 5 MB should be hosted elsewhere (for instance a dedicated audio bucket or a Vimeo unlisted upload) and linked from the lesson tables, to keep the repository light.

## Forward compatibility with static site generators

The directory layout, the YAML frontmatter, and the Markdown body are deliberately compatible with Eleventy and Astro. If at some point a generator is preferred over the current `build.py`, neither the URL structure (`/lessons/lesson-NN.html`) nor the content model needs to change. The generator can be added without breaking existing links or rewriting the lesson source files.

## Linguistic content

All vocabulary and example sentences currently in `content/` are placeholders, structurally plausible but not verified Sofia-Erli data. They are tagged with HTML comments in the generated tables so reviewers can spot them at a glance. They will be replaced with material checked against Kiril Kostov's grammar of Sofia-Erli and against field recordings.

Please do not cite the present placeholder forms as evidence of Sofia-Erli usage. They exist only to exercise the page layout.
